## 🔑 重要決策與發現

### 1. 為什麼選 GitHub + Streamlit 而不是 Railway / Oracle VM
- 完全免費（公開 repo Actions 無限制）
- 不需信用卡驗證
- 程式改動可接受
- 缺點是持倉備份要手動 JSON

### 2. 為什麼需要 IV/RV 雙軌
- yfinance **不提供歷史 IV 回填**
- 必須每天當下抓取累積，至少 30 天才有 IV Rank
- 用既有股價算 RV Rank 當過渡期替代

### 3. 為什麼需要「雲端 URL 優先」邏輯
- 早期上傳時把 `cache/us_daily.parquet` 也推到 repo 內
- 之後 GitHub Actions 寫的是 Releases，而非 repo 內
- 原邏輯「本地有檔案就用本地」導致永遠用舊版

### 4. 為什麼要 session_state
- Streamlit 互動 = 整頁 rerun
- 沒存 session_state 的話，按完查詢按鈕後拖 slider 結果會消失
- 凡是「按一次載入大量資料、之後做互動」的場景都要套這個模式

### 5. 為什麼整合成頁籤
- 4 個 expander 並列容易很長、滾動疲勞
- Tabs 一次只看一個，視覺乾淨且像 Excel sheet

### Day 3（2026-06-01 下午 — Bug 修復與雲端強化）

#### 22. Bug：GitHub Actions 每次都「18 天未更新」強制全量刷新
**症狀**：每天排程跑 `fetch_cache_us.py` 看 log 都顯示「快取已 18 天未更新（≥7），強制全量刷新」，耗時 90 秒。

**根因**：Actions checkout 拿到的是 repo 內早期手動上傳的 `cache/us_daily.parquet`（524 檔、最後日期 2026-05-14）。腳本算 stale 看的是這份舊檔的最後日期 → 18 天 → 強制全量。新版 parquet 上傳到 Releases 但**沒 commit 回 repo**，下次跑還是讀到同一份舊檔。

**修法**：
- `fetch_cache_us.py` 新增 `sync_latest_from_releases()`：執行前先從 `PARQUET_URL`（GitHub Releases）下載最新版覆蓋本地檔，再進入增量邏輯
- `.github/workflows/fetch.yml` 加 `env: PARQUET_URL: ...` 餵給腳本
- 安全機制：若本地版日期 ≥ 遠端版會保留本地（避免回退）

**驗證**：log 出現「🌐 從 Releases 拉取最新 parquet」+「↳ 本地 X / 遠端 Y，採用遠端版」，「既有快取」變成 1012 檔最新日期，改走增量下載。

#### 23. Bug：期權頁面所有合約都標「流動性差」
**症狀**：NVDA 即使熱門 strike（OI 應有上千）都顯示 ❓ 流動性差。

**根因**：yfinance 的 `openInterest` 欄常 lag 一天或直接回 0/NaN，光看 OI < 100 會誤殺。

**修法**：`label_contract` 改為「OI < 100 **且** 今日量 < 20」才標流動性差；`risk_checklist` 第 4 項也加 volume fallback（OI 偏低但量 ≥50 時降一格警示）。

#### 24. yfinance Rate Limit 應對：curl_cffi + 多 API fallback
**問題**：雲端 Streamlit Community Cloud 共用 IP 被 Yahoo 整批限流，期權鏈完全抓不到（YFRateLimitError: Too Many Requests）。

**嘗試一**（curl_cffi）：用 `curl_cffi.Session(impersonate="chrome")` 偽裝真實瀏覽器 TLS 指紋 → **無效**，因為 Yahoo 認的是 IP 不是 UA。

**嘗試二**（多 API fallback）：依序試過三家免費 API
| API | 期權鏈是否免費 | 結果 |
|-----|-------------|------|
| Tradier Sandbox | 需先開美國證券戶（KYC、Foreign Diligence）| 放棄 |
| MarketData.app | 免費版只給股票報價，期權要 Trader $100/月 | HTTP 402 |
| Finnhub | 免費版只給基本資料，期權鏈是 Premium | HTTP 403 |
| Polygon.io | Options 要 $29/月 | 沒試 |

**結論**：免費 + 雲端 + 期權鏈 ≈ 不存在。所有供應商把這個當吸金產品。

#### 25. 加強重試版：磁碟快取 + 智能冷卻 + 過期 fallback
既然找不到可用的雲端 API，全力榨乾 yfinance 的偶發成功機會。

`options_data.py` 加：
- **6 小時磁碟快取** — 抓到的鏈存到 `cache/options/{ticker}_{exp}.parquet`，下次優先讀磁碟
- **智能冷卻** — 403/402 鎖 1 小時（永久鎖類錯誤）、429 鎖 5 分鐘（暫時限流）
- **yfinance 重試** — 失敗等 3s 再試 1 次
- **過期 fallback** — 全失敗時用磁碟舊資料，UI 顯示「資料 X 小時前」橫條
- **強制刷新按鈕** — 清磁碟快取 + 解除冷卻，主動繞過

UI 加：
- 冷卻狀態 caption（顯示哪些 source 還在冷卻幾秒）
- 過期警示橫條（黃色，附使用建議）
- 友善降級訊息：若三來源都鎖，提示「請本機執行」並附指令

#### 26. MarketData IV 缺失反推（已寫好但因 chain 端點 402 用不到）
MarketData.app 的 cached 端點在盤後不回 IV，會讓 Delta 變 step function 而無 ⭐ 推薦標籤。

加 `implied_vol_from_price()` 用 Newton 法從 lastPrice 反推 IV（含合理區間 5%-300% 過濾異常）。雖然 chain 端點本身付費鎖死、此修補目前無法觸發，但留著當未來付費版啟用或 API 規則變化時的保險。

#### 27. IV 反推升級為通用層 + 盤後提示橫條
**問題**:加強重試版部署後使用者實測,雲端 yfinance 在台灣下午時段(美股盤後)雖然有時抓得到鏈,但回傳的 Bid/Ask、IV、OI 全是 0,導致 Delta 算出來變 step function(1.0 / 0.0),沒有 ⭐ 推薦標籤,使用者誤以為是 bug。

**根因**:盤後沒人掛單 → yfinance 沒有有效 Bid/Ask → IV 無法計算 → 整條 Greeks 退化。這跟 yfinance 對 OI 的 lag bug 是兩件事但症狀疊加。

**修法**:
1. **IV 反推搬到 `enrich_chain` 通用層** — 原本只在 `_md_option_chain` 裡跑(MarketData 專用),現在所有資料源(含 yfinance)都會走一次。合理區間 5%-300% 外的 IV 自動從 `mid` 反推,Greeks/Delta/⭐ 標籤系統得以保留。
2. `df.attrs["iv_was_repaired"]` 旗標 — UI 知道這次資料有沒有走反推。
3. 新增 `is_us_market_open()` + `us_market_status_msg()` — 偵測美股是否在 09:30-16:00 ET 交易時段,休市時段(週末/盤前/盤後)各對應一條訊息。
4. UI 期權頁頂部加藍色提示橫條 — 只在休市時段出現,白話解釋「Bid/Ask=0、IV 缺失、OI=0 是 yfinance 對盤後資料的限制,不是 bug」,並告知反推誤差約 5-15%。
5. UI 結果上方加透明度 caption — 有走反推就顯示「🔧 已用 Black-Scholes Newton 法從中價反推」。

**結果**:盤後查 NVDA 也能看到 ⭐ 推薦標籤(雖然 IV 是反推的、有誤差),使用者體驗從「以為壞了」改成「知道是盤後限制 + 系統盡力提供可用資料」。

#### 28. 結論:雲端定位明確
- ✅ **雲端**：選股、大盤狀態、回測、持倉、教學頁籤 — 全部正常
- 🎲 **雲端期權瀏覽**：碰運氣，yfinance 有時通有時不通；磁碟快取讓「曾經通過」的鏈持續可用 6 小時
- 🏠 **本機**：完整功能（含期權瀏覽），yfinance 從家用 IP 不被擋
