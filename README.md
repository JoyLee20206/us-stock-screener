# 🇺🇸 美股量化選股終端機

這是一個自己用的美股選股工具,**完全免費**,跑在網頁上(Streamlit)。你打開網址、按一下「執行掃描」,它就會幫你從一千多檔美股裡篩出「現在最有機會的股票」,還會順便告訴你該配什麼期權合約、停損該放哪、什麼時候該跑。

裡面塞的策略主要學自三個交易大師:**William O'Neil(CAN SLIM)、Mark Minervini、IBD 籌碼分析**。他們的共通信念是:**先看大盤臉色 → 挑強勢股 → 找突破點進場 → 嚴守停損**。

## 它能幫你做什麼

1. **每天告訴你大盤現在能不能進場** — 紅黃綠燈,一眼看完
2. **從一千多檔股票裡挑出最有機會的幾檔** — 三種心法可切換(突破/回檔/綜合)
3. **算好停損、目標價、風險報酬比** — 不用自己手動算
4. **幫你選期權合約**(如果你想用期權做槓桿)— 哪個 strike 最划算、哪個是地雷,直接標出來
5. **追蹤你的持倉**(股票 + 期權)— 有風險自動跳警示
6. **回測歷史掃描結果** — 看看以前選出來的股票後來表現如何

## 怎麼用

- **網頁版**(雲端):https://us-stock-screener.streamlit.app
- **本機版**:電腦下載程式,跑 `streamlit run us_screener_ui.py`

> ⚠️ **重要提醒:雲端版的「期權瀏覽」功能不太穩**
>
> 期權鏈的資料雲端抓不太到(Yahoo 把雲端 IP 列黑名單了,免費的替代資料 API 又都鎖在付費版)。
> 我已經做了一堆補救(磁碟快取、自動重試、過期資料 fallback),但雲端**有時候還是會撈不到**。
>
> **最穩的做法是本機跑**:在你家裡電腦執行,Yahoo 不擋家用 IP。其他功能(選股、回測、持倉等)雲端都正常,只有期權瀏覽有差。

## ✨ 選股功能怎麼運作?

### 1. 進場前先看大盤的臉色(CAN SLIM 的 M)

O'Neil 名言:**「逆勢操作,連神仙都救不了你」**。所以進場前先看一眼大盤紅綠燈:

- 🟢 **多頭健康**:可以積極選股、加碼
- 🟡 **警示**:派發日累積中(機構在偷偷出貨),建議降低持股
- 🔴 **空頭/弱勢**:跌破年線,建議空手觀望

紅綠燈是怎麼判斷的?簡單講就是看四個東西:**SPY 是否站上 200 天均線 + 50 天均線 + 50 天均線方向往上 + 最近 25 天「派發日」少於 4 天**。「派發日」= 大盤下跌超過 0.2% 且成交量超過平均,機構在賣的訊號。

### 2. 三種心法,看你今天想幹嘛

| 心法 | 適合什麼樣的盤 | 簡單講就是... |
|------|--------------|--------------|
| 🚀 **BREAKOUT 突破** | 大盤強、想抓飆股 | 找剛突破近 20 天高點 + 成交量爆 1.5 倍的股 |
| 🎣 **PULLBACK 回檔** | 趨勢明確、想在拉回找買點 | 找站穩月線、稍微回檔但還沒跌破的強勢股 |
| 📊 **COMBO 綜合** | 想全面掃 | 兩種都看,用計分排序 |

### 3. 九個「加分指標」(類似考試評分)

每檔股票最多拿 9 分,通過的條件越多分越高。重要的幾個:

- **VCP 收縮型態**(Minervini 招牌):股價橫盤但波動越來越小,像彈簧蓄能,常是噴出前夕
- **Stage 2 Breakout**:盤整 30 天後突破新高,典型啟動訊號
- **Power Day**:單日跳空 +5% 配上量爆 2 倍 → 機構強力進場印章
- **IBD RS Rating**:把每檔股票過去一年表現拿來互相 PK,排個 1-99 名次,> 80 算頂尖
- **U/D Volume Ratio**:上漲日和下跌日的成交量比。> 1.25 表示機構在偷偷吃貨,< 1.0 表示在派發

### 4. 「硬條件」過濾(沒過直接踢)

不是分數高就好,有些基本門檻一定要過:

- 距離 52 週新高在 25% 之內(沒近最高點就不夠強)
- 短中長期均線多頭排列(MA5 > MA20 > MA60)
- 季線(MA60)還在上揚(不是橫盤死水)
- 股價 ≥ $10、日成交額 ≥ $5M(避免流動性陷阱)

### 5. 進場前算好「會賠多少、想賺多少」

| 指標 | 算法 | 拿來幹嘛 |
|------|------|---------|
| 建議停損 | max(現價 - 2.5×ATR, MA20×0.99) | 跌到這就跑,別問為什麼 |
| 建議目標 | 現價 + 風險金額 × R/R 倍數(預設 2) | 賠 1 賺 2,期望值要正 |
| ATR% | 股票平均日波動 | 太高代表股價跳很兇,要小心 |
| 風險% | 進場價到停損的距離 | 看你能不能接受 |

### 6. 持倉管理(像個小 Bloomberg)

#### 📈 股票持倉
新增一筆持倉後,系統會幫你自動算:報酬%、距停損%、距目標%、目前 RS Rating 還剩多少。並且**自動跳警示**:

- 🔴 嚴重:**跌破停損** / 跌破 MA50
- 🟡 一般:RS 掉到 50 以下 / 量大長黑 / 8 週都沒漲 5% 以上(代表浪費資金)
- 🟢 正面:創 30 日新高 + 量增 = 加碼訊號 / 達到目標價

**Minervini 鐵則**:不管你輸入什麼停損,系統會自動緊縮成「進場價 × 0.93」(-7%)和你輸入的之間取較緊那個。沒辦法設更寬鬆,這是寫死的紀律。

#### 🎯 期權持倉
你買的每口 Call/Put 都會即時抓報價、算 Greeks(Delta/Theta 等),自動警示:

- 🚨 **DTE ≤ 7 天**:時間價值崩塌,趕快平倉
- 🔴 **跌 -50%**:觸發停損
- 🟢 **賺 +100%**:可以考慮獲利了結
- 💎 **已 ITM**(內含價值大於 0)
- ⚡ **時間價值快燒完**:Theta 加速吃
- 📅 **跨財報日**:IV crush 風險

### 7. 績效回測(看以前的選股有沒有用)

每次掃描完系統會自動存一份快照到 `scans/` 資料夾。累積 5-12 週後,「績效回測」頁籤就會告訴你:

- 過去選出來的股票,1 週後 / 4 週後 / 12 週後**平均報酬多少、勝率多高**
- 哪些訊號(VCP / Stage2 / Power Day)後來表現特別好
- 分數越高的股票後來表現是不是真的越好

如果發現「8-9 分」的勝率才 40%,那就要重新檢討策略。

## 🎯 期權工具(純買方視角,新手友善)

這個部分是為了**想用期權做槓桿但又不想踩死亡陷阱**的人設計的。買方統計上 70-80% 歸零,主要死因是:**選錯 strike、漏看財報、進場 IV 太貴**。系統會把這三個地雷都標出來。

### 1. 📚 期權新手教學
App 內直接渲染一份 280 行的新手指南。從零講到能看懂 Greeks、會避開常見陷阱。第一次用先點開來掃一遍。

### 2. 🎯 期權瀏覽(核心功能)

輸入代號(例如 NVDA)→ 選到期日 → 按查詢,你會看到:

- **頂部摘要列**:標的現價、剩幾天到期、Call/Put 各幾筆、**IV 算貴算便宜、下次財報日**
- **⭐ 推薦合約卡**:系統替你挑好的「最甜蜜點」合約(Call 和 Put 各一張)
- **跨財報時頂部紅色橫條**:警告你這個合約會跨過財報,IV crush 風險超高
- **Call 鏈 / Put 鏈表格**:每口合約都有智能標籤、Greeks、盈虧平衡

### 3. 🏷️ 智能標籤(9 種,一眼判斷好壞)

每口合約系統都會貼一個標籤,優先序由上往下:

#### ✅ 可以考慮的
| 標籤 | 條件 | 給誰用 |
|------|------|--------|
| ⭐ 推薦 | Δ 0.45-0.65 + 21-45 天到期 | **新手首選**(性價比甜蜜點) |
| 💎 ITM 穩 | Δ > 0.70 | 想用期權替代買股票、要高中獎率 |
| 🟠 略 ITM | Δ 0.65-0.70 | 保守派,稍貴但穩 |
| ⏱️ DTE 不對 | Δ 對了但天數不對 | 還可以,但時間軸要重挑 |
| 🟡 略 OTM | Δ 0.30-0.45 | 槓桿大、但中獎率掉 |

#### ❌ 應該避開的
| 標籤 | 條件 | 為什麼避 |
|------|------|---------|
| ⚠️ 太 OTM | Δ < 0.30 | 90% 機率歸零 |
| 🔥 高 IV | IV > 50% | 進場太貴,財報後 IV crush 直接賠光 |
| 💀 Theta 黑洞 | DTE < 14 天 | 每天時間價值蒸發 5-10% |
| ❓ 流動性差 | OI < 100 **且**今天量 < 20 | 想出場沒人接 |
| 🚨 跨財報 | 合約到期前有財報 | IV crush 風險 |

(註:6/1 修正了「流動性差」誤判 bug,以前單看 OI 會把熱門合約也錯標)

### 4. 🔍 合約分析(挑好合約再用這個深入看)

下拉選一口合約,系統會給你三樣東西:

#### ✅ 進場前 7 項風險清單
紅黃綠燈逐項檢查:DTE、Delta、IV、OI、Volume、Bid-Ask 價差、跨財報日。
整體判定**綠燈 GO / 黃燈 CAUTION / 紅燈 STOP**。**要全綠才下單**。

#### 💡 名詞解釋(可展開)
看不懂 Δ / Θ / IV / BE 是什麼?點開有完整白話定義。

#### 🎮 What-if 模擬器(我的最愛之一)
3 個 slider:
- **股價變動 %**:假設股票漲跌多少
- **天數經過**:假設過了 N 天
- **IV 變動 %**:假設 IV 漲跌多少(例如財報後常見 -30%)

拖一拖就看到「**這口合約現在值多少、賺多少、賠多少**」。**強烈建議在進場前玩一下**,你會看到 Theta 怎麼吃你的獲利、IV crush 多可怕。

#### 📈 到期日損益曲線(Plotly 互動圖)
X 軸是到期日股價、Y 軸是損益。曲線會自動標出:
- 🟠 現價
- 🔵 行權價(K)
- 🟢 盈虧平衡點(BE)
- 🔴 最大虧損(就是你花的權利金)

圖下面有「📖 怎麼看這張圖?」可以展開,**用你選的這口合約的實際數字**逐段解釋。

### 5. 📈 對掃描結果批次產生期權建議
選股完成後,下面有個「對 Top N 名產生期權建議」按鈕。一次幫你看 5-10 檔的 ⭐ 推薦合約,還能勾「**避開財報**」自動挑財報後到期。

### 6. 📊 IV 算貴還算便宜?(雙軌設計)

買期權最怕的就是「IV 太貴進場 → 財報後 IV 崩塌 → 賠光」。所以要有指標告訴你**現在的 IV 在歷史上算什麼位置**:

| 指標 | 什麼時候用 | 怎麼讀 |
|------|----------|--------|
| **IV Rank**(真正準) | 系統累積 IV 滿 30 天後才會出現 | ≥ 70% 🔥 偏貴 / 30-70% 中等 / < 30% ✅ 便宜 |
| **RV Rank**(估算,立即可用) | 累積還沒滿時用這個替代 | 一樣讀法 |

(yfinance 不提供歷史 IV,所以要每天自己抓累積。GitHub Actions 每天 5:45 跑一次。)

### 7. 📅 財報日警示
- 系統會幫你抓每檔股票的下次財報日
- 跨財報的合約自動標 🚨,頂部出紅色橫條
- 期權持倉如果跨財報,還會警示「**財報倒數 N 天,建議平倉避 IV crush**」

### 8. 📈 IV 暴衝偵測(財報前 1-3 週很常見)

財報前大家搶買期權避險 → 把 IV 推到不合理的高 → 這時候進場最慘。系統用 **IV / RV 比例**量化這個現象:

| IV/RV | 等級 | 含義 |
|-------|------|------|
| < 1.20 | ✓ 正常 | 可以進場 |
| 1.20-1.40 | 📈 偏高 | 已有事件溢價,要謹慎 |
| > 1.40 | 🔥 暴衝 | **不要進場**,等財報後再說 |

只有在「**IV/RV 偏高 + 財報在 30 天內**」才會確認為財報溢價警報,單純 IV 偏高不會誤觸發。

### 9. 📆 一鍵切換到「財報後到期」

跨財報的合約頂部會出現按鈕:
```
🚨 跨財報警示:下次財報 2026-08-27(5 天後)... [📆 改選 2026-09-18]
                                                   ↑ 按一下自動切財報後 14 天最接近的到期
```

批次推薦那邊也有「避開財報」toggle,打開後系統自動幫每檔挑財報後到期。沒財報的標的(像 ETF)自動退回 30 天邏輯,不會出錯。

## ☁️ 雲端期權瀏覽為什麼有時候不穩?

(這段是 2026-06-01 新加的章節,因為這天踩到一堆坑)

### 問題本質
**Streamlit Cloud 用一群共用 IP**,Yahoo 把這些 IP 整批列限流名單(`YFRateLimitError: Too Many Requests`)。我們系統性試過**所有免費期權 API**:

| 供應商 | 期權鏈免費嗎? | 實測結果 |
|--------|-------------|---------|
| Yahoo(yfinance) | ✅ 但雲端 IP 被擋 | 雲端常 429 |
| MarketData.app | ❌ 要 Trader $100/月 | HTTP 402 Payment required |
| Finnhub | ❌ 要 Premium $25+/月 | HTTP 403 You don't have access |
| Polygon.io | ❌ Options 要 $29/月 | 沒實測 |
| Tradier Sandbox | ❌ 要先開美國證券戶 | 申請流程繁瑣放棄 |
| **你的券商 Firstrade** | ❌ 根本沒公開 API | — |

期權鏈是供應商的**吸金產品**,免費版幾乎都沒有。

### 雲端能撐的補救機制(已內建)

雖然 yfinance 在雲端常失敗,還是想盡辦法讓它**偶爾通的那次撐久一點**:

| 機制 | 做什麼 | 比喻 |
|------|--------|------|
| **6 小時磁碟快取** | 抓到一次成功就存起來,6 小時內不再打 API | 抓到一次魚冰起來吃半天 |
| **智能冷卻** | 403/402 鎖 1 小時、429 限流鎖 5 分鐘 | 已知會打不通的不要狂打 |
| **yfinance 自動重試 1 次** | 失敗等 3 秒再試一下 | 偶發網路抖動可吸收 |
| **過期 fallback** | 全部失敗時用磁碟上的舊資料 + 黃色橫條告訴你 | 沒新鮮的就吃罐頭 |
| **🔄 強制刷新按鈕** | 清掉所有快取和冷卻、重抓 | 你想吃新鮮的就按 |
| **curl_cffi 偽裝 Chrome** | 假裝是真實瀏覽器去打 yfinance | 對 IP 限流沒用,但對 UA 限流有用 |
| **冷卻狀態提示** | UI 顯示「⏳ yfinance 還要 287 秒解除冷卻」 | 透明,你知道在等什麼 |
| **降級提示** | 三來源都掛時,直接告訴你「請本機執行」 | 不繞圈圈 |
| **IV 反推(通用層)** | yfinance 盤後 IV=0 時,從中價用 Newton 法回推 IV,讓 ⭐ 推薦標籤、Delta 仍可用 | 沒體溫計就用手摸額頭估 |
| **盤後提示橫條** | 自動偵測美股是否休市,休市時頂部顯示藍色橫條解釋「為什麼 Bid/Ask、IV、OI 都是 0」 | 告訴你那不是壞掉,是現在沒人交易 |

### 本機跑就完全沒問題

家用 IP 不在 Yahoo 限流名單,本機跑 yfinance 期權鏈一切正常。

```powershell
cd "你的專案目錄"
streamlit run us_screener_ui.py
```

## 📦 自己要怎麼部署一套?

### Step 1:在 GitHub 開一個 Public Repo
名字隨便,例如 `us-stock-screener`。**一定要 Public**,Private 的 Actions 有限額。

### Step 2:把檔案上傳上去
網頁拖拉:`https://github.com/你的帳號/你的 repo/upload/main`
要傳的:`us_screener_ui.py`、`fetch_cache_us.py`、`fetch_iv_history.py`、`options_data.py`、`期權新手指南.md`、`requirements.txt`、`README.md`、`.gitignore`、`.github/workflows/fetch.yml`、`.github/workflows/fetch_iv.yml`

**`requirements.txt`** 內容:
```
streamlit
pandas
yfinance
pyarrow
openpyxl
requests
plotly
curl_cffi
```
(2026-06-01 加 `curl_cffi`,給雲端 yfinance 偽裝瀏覽器用)

### Step 3:第一次手動觸發兩個排程
- Actions 頁面 → "Daily US Stock Cache Update" → Run workflow
- Actions 頁面 → "Daily IV History Snapshot" → Run workflow

等 5 分鐘,去 Releases 頁面確認 `us_daily.parquet` 和 `iv_history.parquet` 都上去了。

### Step 4:拿你的 Parquet URL
複製這串(把 `YOUR_USERNAME` 改成你的 GitHub 帳號):
```
https://github.com/YOUR_USERNAME/us-stock-screener/releases/download/data-cache/us_daily.parquet
```

### Step 5:Streamlit Cloud 部署
1. 去 [share.streamlit.io](https://share.streamlit.io) 用 GitHub 帳號登入
2. 點 **New app** → 選你的 repo / branch: main / Main file: **`us_screener_ui.py`**
3. **Advanced settings → Secrets** 貼這段(URL 換成你的):
   ```toml
   PARQUET_URL = "https://github.com/YOUR_USERNAME/us-stock-screener/releases/download/data-cache/us_daily.parquet"
   ```
4. 按 **Deploy**

### Step 6:(關鍵)workflow 也要設 PARQUET_URL

不然 Actions 每次跑都會以為快取很舊強制全量(90 秒,浪費)。
編輯 `.github/workflows/fetch.yml` 在「Run fetch_cache_us.py」那步加 env:

```yaml
- name: Run fetch_cache_us.py
  env:
    PARQUET_URL: https://github.com/${{ github.repository }}/releases/download/data-cache/us_daily.parquet
  run: python fetch_cache_us.py
```

這樣 Actions 跑的時候會先從 Releases 拉最新版,改走快速的增量下載。

## 🔧 常見問題

**Q:雲端版的掃描結果 / 持倉 重啟後就消失了?**
A:雲端的檔案系統是**暫時的**,App 休眠或重啟就清空。請:
- 持倉:側邊欄的「📥 下載持倉 JSON」每次新增後備份一下,下次用「📤 還原」貼回
- 掃描結果:用 Excel 報表存下來,或下載 `scans/*.parquet` 留檔

**Q:為什麼快取顯示舊資料、找不到我新加的 ETF?**
A:點 App 頂部右側的「🔄 刷新雲端資料」按鈕,Streamlit 預設快取 1 小時。

**Q:IV Rank 怎麼一直顯示「估算(RV Rank)」?**
A:yfinance 不提供歷史 IV,要每天自己抓累積,要累積 30 天才有意義。期間先用「實現波動率」算的 RV Rank 替代,讀法一樣。等滿 30 天會自動切換為真的 IV Rank。

**Q:期權瀏覽找不到財報日?**
A:ETF(SPY、QQQ)沒財報是正常的,系統會顯示「無資料」。某些小型股 yfinance 可能拿不到,也會顯示「無資料」。

**Q:IV/RV 比例突然飆到 1.5 是什麼意思?**
A:市場已經在 pricing in 即將到來的事件(通常是財報)。**這種時候進場買方最容易踩 IV crush**。建議:等財報後 IV 回穩再進、或用「📆 改選財報後到期」一鍵切到財報後合約。

**Q:「避開財報」開關打開後,沒財報的標的會怎樣?**
A:自動退回一般 30 天到期日邏輯,跟關閉時一樣。所以同時掃有財報股 + ETF 不會出問題。

**Q:GitHub Actions 排程會自動跑嗎?**
A:會。Public repo 排程 workflow **免費無限次**。每天清晨 5:30 + 5:45 自動跑。Repo 閒置 60 天會被暫停(手動觸發任何 workflow 或 push 一個 commit 就解除)。

**Q:(2026-06-01)為什麼以前 Actions log 每次都顯示「18 天未更新強制全量」?**
A:Bug。Actions checkout 拿到的是 repo 內早期手動上傳的 parquet(5/14 那份),腳本算 stale 一直算成 18 天,所以每次都強制全量 90 秒。已修:加了「執行前先從 Releases 拉最新版」邏輯,workflow 也加了 env: PARQUET_URL。現在每次都改走 5 秒的增量下載。

**Q:(2026-06-01)為什麼期權頁面所有 strike 都標「流動性差」?**
A:Bug。yfinance 的 `openInterest` 常 lag 一天或回 0,光看 OI < 100 會把熱門合約也錯標。已修:改成「**OI < 100 且 今日量 < 20**」才標流動性差,風險清單第 4 項也加 volume fallback。

**Q:(2026-06-01)雲端期權瀏覽抓不到怎辦?**
A:yfinance 在 Streamlit Cloud 共用 IP 常被 Yahoo 限流。已內建一堆補救機制(磁碟快取 + 智能冷卻 + 過期 fallback + 重試 + 偽裝瀏覽器),但有時候還是會全失敗。處理順序:

1. 按「🔄 強制刷新」清快取重抓
2. 看冷卻 caption 顯示剩多少秒,等冷卻結束再試
3. 如果出現黃色橫條「資料 X 小時前」,代表用過期快取撐著,內容可以參考但 Bid/Ask 可能不準
4. **最穩做法是本機跑** `streamlit run us_screener_ui.py`(家用 IP 不被擋)

**Q:(2026-06-01)既然 IV 能反推,OI 為什麼不能?**
A:**本質不同**。IV 是「黑箱裡的一個數學參數」,從 Black-Scholes 公式跑出來,給定股價、行權價、剩餘天數、利率、中價,就能用 Newton 法逆推 σ。OI 是「**全美國目前有幾個人正在持有這口合約**」的**會計事實**,只能去問 OCC(美國期權結算公司),沒有任何資料能反推。

打個比方:
- IV 反推像「知道一杯飲料的甜度,反推糖加了多少」(配方明確,能算)
- OI 像「美國目前有幾人正在持有這口合約」(只能查統計,沒公式)

**但其實 OI=0 影響很小**,因為系統用 Volume 當代理指標:

```python
# 流動性差判斷:OI < 100 「且」 Volume < 20 才標
if oi < 100 and vol < 20:
    return "❓ 流動性差"
```

看你看到的:$215 strike OI=0 但 Volume=1889(週五全天累積)→ 系統正確判定**不是**流動性差。OI 的 lag 被 Volume 蓋過去了。

用到 OI 的地方對照表:
| 用到 OI 的地方 | OI=0 時影響 |
|--------------|-----------|
| 智能標籤「❓ 流動性差」 | ✅ 用 OI + Volume 雙條件,不受影響 |
| 風險清單第 4 項 | ⚠️ 紅黃綠燈會偏紅,但已加 Volume fallback |
| ⭐ 推薦合約卡 | ✅ 不使用 OI(只看 Delta + DTE) |
| What-if 模擬器、損益曲線 | ✅ 完全不用 OI |
| 期權持倉警示 | ✅ 不依賴 OI |

**真的想要 OI 三個辦法**(目前都沒做):
1. 自己每天美股收盤後抓 OI 累積成 history(類似 IV history 機制)— 工程量中,但反正都是 lag 資料,意義有限
2. 訂 IBKR/Polygon 等付費資料源($29+/月)
3. 接受並透過 Volume 判斷流動性 ← **現在的做法**

**Q:(2026-06-01)為什麼我下午查 NVDA,Bid/Ask、IV、OI 全都是 0?**
A:**這是正常的,不是 bug**。台灣下午時段美股完全休市(美東半夜),具體現象:
- Bid/Ask=0:沒人掛單(休市)
- IV=0:沒有買賣價算不出 IV(yfinance 無法計算)
- OI=0:yfinance 對 OI 的更新本來就慢半拍,休市時更不會更新
- Volume:會顯示上次交易日(週五)的累積量

**系統已自動補救**:偵測到美股休市時,頁面頂部會出現藍色提示橫條告訴你這件事,並啟用 IV 反推(從中價用 Newton 法回推),讓 ⭐ 推薦標籤、Delta、Theta 仍能運作。但反推值會有 5-15% 誤差。**想看最準資料請在台灣晚上 21:30 美股開盤後查詢**。

**Q:(2026-06-01)為什麼免費版的 Finnhub / MarketData 一直 403 / 402?**
A:期權鏈是這兩家的吸金產品,免費版完全不開放。Finnhub 要 Premium($25+/月)、MarketData 要 Trader($100/月)才解鎖。我們程式碼留著 token 接口,真要用就升級。Streamlit Secrets 不設這兩個 token 完全不影響(只走 yfinance)。

**Q:我的券商 Firstrade 有沒有 API 可以用?**
A:**沒有官方 API**。Firstrade 從來沒發佈過給散戶用的 REST API。GitHub 上有非官方的爬蟲(用帳號密碼登入),但**違反 ToS、不穩、Firstrade 改版就壞、抓不到完整期權鏈**。即使 IBKR / Schwab 等有官方 API 的券商,**期權即時報價也要另外訂 OPRA Data feed($4.5-30/月)**,API 跟市場資料權利是分開的。所以**沒有便宜的雲端解法**,本機跑 yfinance 仍是最划算。
